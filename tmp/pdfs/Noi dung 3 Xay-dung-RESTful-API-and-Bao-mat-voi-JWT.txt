try {
// 1. Validate input

if (!title || title.length < 10) {
return res.status(400).json({
status: 'error',
message: 'Tiêu đề phải có ít nhất 10 ký tự',

};

Bảo mật & Xác thực với JWT
Phần quan trọng nhất: Làm sao để bảo vệ API khỏi truy cập trái phép?
Vấn đề của Session truyền thống
Cơ chế Session/Cookie
Khi user đăng nhập, Server tạo một Session ID và lưu vào Database hoặc
Memory. Session ID này được gửi về Browser dưới dạng Cookie. Mỗi request sau
đó, Browser tự động gửi Cookie lên, Server kiểm tra Session ID để biết user là ai.
Nhược điểm
• Khó Scale: Khi có nhiều server, phải đồng bộ Session giữa các server
• Không phù hợp Mobile: Mobile app không hỗ trợ Cookie tốt
• Tốn tài nguyên: Phải lưu Session trong Database/Memory
Kết luận: Session phù hợp cho Web app đơn giản, nhưng với RESTful API phục vụ đa nền tảng (Web + Mobile), JWT là giải pháp tốt hơn.
JWT (JSON Web Token) - Tấm vé thông
hành số
JWT là gì?
JWT là một chuỗi mã hóa chứa thông tin user (id, role, email) được ký bởi Server. Client
giữ token này và gửi kèm mỗi request. Server chỉ cần verify chữ ký mà không cần truy vấn
Database.
Ưu điểm của JWT
• Không cần lưu Session trên Server (Stateless)
• Dễ dàng Scale ra nhiều server
• Phù hợp cho cả Web và Mobile
• Có thể tự chứa thông tin user (self-contained)

Một token JWT gồm 3 phần, ngăn cách nhau bởi dấu chấm (.). Mỗi phần được encode bằng Base64.

Sau khi đăng nhập và nhận được token, mỗi khi gọi API bảo mật, Client phải gửi kèm token trong Header.
1
Client gửi Request kèm Token

Headers:
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
2
Middleware chặn và kiểm tra Token
Trước khi vào Controller, Middleware "auth" sẽ verify token bằng secret key. Nếu hợp lệ, giải
mã payload để lấy thông tin user.

Bảo mật nâng cao: Refresh Token
Vấn đề: Access Token có thời gian sống ngắn
Để bảo mật, Access Token thường có exp (expire time) chỉ 15 phút hoặc 1 giờ. Nhưng
việc bắt user đăng nhập lại liên tục rất khó chịu.
Giải pháp: Dùng Refresh Token
Khi đăng nhập, Server trả về 2 token:
• Access Token: Thời gian ngắn (15 phút), dùng để gọi API
• Refresh Token: Thời gian dài (7 ngày hoặc 30 ngày), lưu trong Database, dùng để
xin cấp lại Access Token mới
Khi Access Token hết hạn, Frontend gọi POST /api/refresh với Refresh Token để
lấy Access Token mới mà không cần user đăng nhập lại.
